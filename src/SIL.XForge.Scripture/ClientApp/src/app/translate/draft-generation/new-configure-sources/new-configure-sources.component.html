This is an experimental simplified page for configuring sources. Please leave us feedback on how it works for you. How to configure draft sources

Generating a draft requires two steps. First we need to teach a model how to translate from one language into another. The 'from language' is also known as the 'Source', and the 'to language' is also known as the 'Target'.

Source languages already known to the model, or closely related to the Target language usually work best. Here is a list of the languages that the model knows already.

The current project {{ currentProjectShortName }} will be used to train a model that can translate into {{ targetLanguageDisplayName }}.

Choosing Source Projects for training the model

Choose one or two texts in the 'Source Language'. Sometimes choosing a Back Translation improves the model. It is worth testing with and without the Back Translation to see which setting gives you the best results.

There is an option to include other translation examples in training by uploading a spreadsheet with two columns. The first column contains the Source text and the second column contains its translation in the Target language.

Choosing a Drafting Source

The model will create drafts by translating books from the 'Drafting Source Project'. It is rare that the model will give useful results if the Drafting Source Project isn't included in the Source projects for training the model.

You will be able to look at the drafts before you need to decide how to use them. Scripture Forge will not automatically add the generated drafts to any project. They will only be available in Scripture Forge until you add them to a project.
